# School360 예정사업 관리

Sprint 26 adds planned-project management directly to the School360 screen.
The feature uses the existing `ProjectService` boundary and does not modify the
Connector Framework.

## 사용 방법

- `사업 추가`: 선택한 학교에 예정사업을 직접 등록합니다.
- `수정`: 사업명, 분류, 상태, 예산, 기간, 예상 조달일, 출처, 메모를 수정합니다.
- `삭제`: 확인 후 선택한 사업을 삭제합니다.
- `예정사업 가져오기`: `.xlsx` 파일의 사업을 선택한 학교에 일괄 등록하거나 갱신합니다.
- `양식 다운로드`: 공식 한글 머리글과 예시 데이터가 포함된 양식을 저장합니다.

## Excel 열과 검증

필수 열은 `사업명`입니다. `예정사업명`, `프로젝트명`, `사업 이름`도 자동으로
인식합니다. 선택 열은 `사업 분류`, `상태`, `예산`, `시작 연도`, `종료 연도`,
`예상 조달일`, `메모`, `출처`입니다. 공백과 대소문자를 정규화한 사업명이 같은
경우 기존 사업을 갱신하고, 그렇지 않으면 새 사업을 등록합니다.

- 상태: `진행중`, `예정`, `완료`, `보류`
- 예산: 0 이상의 숫자(쉼표와 `원` 허용)
- 연도: 2000~2200, 종료 연도는 시작 연도 이상
- 예상 조달일: `YYYY-MM-DD`, `YYYY.MM.DD`, `YYYY/MM/DD`, `YYYYMMDD`

행별 검증 오류는 가져오기 요약에 표시되며, 정상 행은 한 트랜잭션에서 저장됩니다.

## 저장 및 호환성

기존 `projects` 테이블에 `expected_procurement_date`와 `memo` 열을 추가하는
후방 호환 마이그레이션을 사용합니다. 기존 프로젝트 레코드와 서비스 호출은 그대로
유지됩니다. 루트 `connectors/` 패키지와 커넥터 수명 주기에는 변경이 없습니다.
